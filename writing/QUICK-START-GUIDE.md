# 電子報創作系統：快速開始指南
> M&A 商業史電子報完整工作流程

---

## 📚 你現在有的三個文件

### 1. **`newsletter-brief.md`** (原始版)
- **用途**：深度寫作哲學與技法
- **何時看**：想深入理解某個寫作技巧時

### 2. **`newsletter-brief-ENHANCED.md`** (強化版) ⭐
- **用途**：完整創作指南，包含所有規範
- **何時看**：開始新案例前、遇到寫作問題時
- **包含內容**：
  - 字數分配（每章節具體字數）
  - 研究檢查清單
  - 事實查核協議
  - 語言風格融合指南

### 3. **`article-production-templates.md`** (生產模板) ⭐⭐⭐
- **用途**：實戰模板，直接複製填空
- **何時用**：每次寫文章時
- **包含模板**：
  - A. 研究檔案
  - B. 文章大綱
  - C. 事實查核清單
  - D. 引用格式
  - E. 語言風格自檢表
  - F. 週度生產時間表

## 🚀 從零開始：你的第一篇文章

### Phase 1: 選題（Day 1-2）

**Step 1: 找近期鉤子**
```
打開 Google News / Bloomberg / Financial Times
搜尋關鍵字：「merger」「acquisition」「M&A」

問自己：
- 這週有什麼大型併購新聞？
- 有哪個產業正在整合？
- 有哪個失敗併購案週年紀念？
```

**Step 2: 連接歷史案例**
```
近期鉤子：_________________

這讓我想到的歷史案例：_________________

為什麼這兩者相關？
_________________
```

**Step 3: 評分（用模板H）**
```
塑造當代生活：___/5
資料可獲得性：___/5
戲劇張力：___/5
深層話題價值：___/5
近期鉤子可行性：___/5

總分 > 18 → 可以寫！
```

### Phase 2: 研究（Day 3-5）

**Step 1: 開新檔案**
```
複製 article-production-templates.md 的「模板A：研究檔案」
儲存為：[案例名稱]-research.md
```

**Step 2: 按順序收集資料**

**第一優先（必須）：**
- [ ] 公司年報（併購前後3年）
  - 去哪找：公司官網 Investor Relations 部分
- [ ] 併購公告/新聞稿
  - 去哪找：公司官網、PR Newswire、Business Wire
- [ ] 監管文件
  - 美國：SEC Edgar（搜尋8-K、S-4表格）
  - 歐盟：歐盟委員會競爭案件資料庫

**第二優先（豐富故事）：**
- [ ] Financial Times / Wall Street Journal 報導
  - 搜尋技巧：用 Google 搜尋「site:ft.com [公司名稱]」
- [ ] Bloomberg 新聞
- [ ] 產業分析報告
  - 去哪找：McKinsey、BCG、Bain 公開報告

**第三優先（如果找得到）：**
- [ ] CEO 專訪、回憶錄
- [ ] 紀錄片、書籍
- [ ] 學術論文（Google Scholar）

**Step 3: 填寫研究檔案**
```
一邊收集資料，一邊填寫模板A
重點：記錄每個資料的來源！

格式：
「交易價格1830億美元」
來源：Vodafone新聞稿，2000-02-03，第2頁
```

**資料收集夠了嗎？判斷標準：**
- [ ] 能回答背景鋪陳的4個問題
- [ ] 有具體的交易數字和結構
- [ ] 知道至少1-2個關鍵轉折點
- [ ] 有併購後的結果數據

---

### Phase 3: 大綱（Day 6）

**Step 1: 複製模板B**
```
從 article-production-templates.md 複製「模板B：文章大綱」
儲存為：[案例名稱]-outline.md
```

**Step 2: 按結構填寫（先不寫完整句子，用關鍵字）**

```
開篇鉤子：
- 近期鉤子：「2026年X月，[公司]併購[公司]」
- 轉折：「但25年前也有一場...」
- 歷史引入：「Vodafone-Mannesmann」
- 重點預告：「1830億怎麼燒掉的」

背景Q1：Vodafone怕被邊緣化、Mannesmann成本惡化
背景Q2：電信自由化、跨國網路競賽
背景Q3：德國市場唯一選擇
背景Q4：監管窗口期6個月

... （依此類推）
```

**Step 3: 確認邏輯順序**
```
讀一遍大綱，問自己：
- 每個部分承上啟下嗎？
- 有沒有突兀的跳躍？
- 高潮在哪裡？（應該在「深層博弈」或「轉折點」）
```

### Phase 4: 初稿（Day 7-8）

**重要心態：初稿不求完美，先寫完！**

**Step 1: 設定環境**
```
- 關閉社群媒體
- 打開研究檔案（一個螢幕）
- 打開大綱（另一個螢幕）
- 準備寫作（新文件）
```

**Step 2: 按大綱逐段寫**

**技巧：先寫骨架，再填血肉**
```
骨架（先寫這個）：
「Vodafone為什麼願意付40%溢價？表面上是因為Mannesmann值得，
但內部郵件顯示真正原因是他們沒有其他選擇了。」

血肉（再加這個）：
「Vodafone為什麼願意付40%溢價？
在當時的電信業併購中，這個數字算高的。

表面上，這是因為Mannesmann『很值得』——
擁有德國最大的移動網路，3000萬用戶，穩定現金流。

但翻開內部郵件（於2002年洩露），你會發現真正的原因：
『如果我們不出手，BT或DT隨時會買走他們。
到時候我們不只失去德國，還會面對更強大的競爭對手。』

這不是『Mannesmann很值錢』，
而是『我們輸不起』。」
```

**Step 3: 每寫500字停下來**
```
用「模板E：語言風格自檢表」快速檢查：
□ 有數據支撐嗎？
□ 太正式嗎？（改口語化）
□ 有告訴讀者「所以呢」嗎？
```

**Step 4: 字數分配提醒**
```
寫到一半時，檢查進度：
- 寫完背景了 → 應該約800字
- 寫完交易細節 → 應該約1800字
- 寫完深層博弈 → 應該約3000字

如果超出太多，砍掉不必要的細節
如果太少，加具體例子
```

### Phase 5: 事實查核（Day 9）

**Step 1: 複製模板C**
```
從 article-production-templates.md 複製「模板C：事實查核清單」
```

**Step 2: 逐項驗證**

**核心事實（找兩個來源互相印證）：**
```
「交易價格1830億美元」
✓ 來源1：Vodafone新聞稿2000-02-03
✓ 來源2：Financial Times 2000-02-04
已驗證！
```

**詮釋性內容（確認有標示）：**
```
原文：「這意味著Vodafone當時壓力很大」
修改：「這可能意味著Vodafone當時壓力很大」
       （分析依據：內部郵件 + 時間窗口壓力）
已標示！
```

**場景重建（確認有說明）：**
```
原文：「那晚的晚餐...」
修改：「據Horn-Smith後來回憶，那晚的晚餐...」
       （來源：Business Week專訪，2005）
已標示！
```

### Phase 6: 潤稿（Day 10）

**Step 1: 禁用詞搜尋**
```
用Ctrl+F（或Cmd+F）搜尋：
- 協同效應 → 改成具體描述「每年節省10億成本」
- 強強聯手 → 刪除，用事實說話
- 優勢互補 → 改成「A有X，B有Y」
- 偉大的 → 刪除或改成具體形容
```

**Step 2: 親和力測試**
```
隨機挑3段，大聲朗讀

聽起來像在和朋友聊天嗎？
□ 是 → OK！
□ 否 → 修改成口語化

例子：
❌「該交易顯著超出產業慣例」
✓「這個價格在當時算很高的」
```

**Step 3: 節奏調整**
```
找出連續3個以上的長句 → 插入短句製造節奏

例子：
「Vodafone的估值模型建立在用戶持續增長的假設上，
但當時歐洲移動電話滲透率已達65%，
增長空間正在快速耗盡。」

改成：
「Vodafone的估值模型建立在用戶持續增長的假設上。
但問題來了：當時歐洲移動電話滲透率已達65%。
增長空間？正在快速耗盡。」
```

**Step 4: 金句檢查**
```
有1-2個讓人印象深刻的句子嗎？

例子：
「當ego大於strategy時，再精密的財務模型都只是童話。」
「輸的人帶著錢離開了，贏的人背著債留了下來。」

如果沒有，試著在結語部分創造一個。
```

### Phase 7: 最終檢查（Day 11）

**完整讀一遍，用模板E完整檢查**

```
□ 字數3000-5000字？實際：_______
□ 有近期鉤子連接歷史？
□ 所有數字有來源？
□ 語調親和但犀利？
□ 有1-2個金句？
□ 有讓人印象深刻的場景？
□ 文末有參考資料列表？
□ 檢查過禁用詞？
□ 大聲朗讀過，聽起來像「聊天」？
```

**加上文末引用列表（用模板D）**

```markdown
---

【本文主要參考資料】

**一手來源：**
- Vodafone年報2000-2005
- Vodafone-Mannesmann併購公告，2000-02-03
- 歐盟委員會併購審查文件，Case No COMP/M.1795

**二手來源：**
- Financial Times, "The deal that destroyed value", 2006-08-12
- Business Week, Horn-Smith專訪, 2005-05-20

**數據來源：**
- Bloomberg歷史股價數據
- GSMA產業報告1999-2005
```

### Phase 8: 發布（Day 12）

**標題最後確認**
```
好標題特徵：
✓ 具體（有公司名或數字）
✓ 有張力（暗示衝突或意外）
✓ 引發好奇

範例：
❌「一場失敗的併購案」（太籠統）
✓「1830億美元怎麼燒掉的：Vodafone世紀豪賭全記錄」
```

**電子報格式**
```
主旨：[標題]

正文：
Hi，

[開篇鉤子]

[正文]

---
【本文主要參考資料】
...

---
下週我們會聊：[預告]

如果你有想了解的M&A案例，歡迎回信告訴我。

Morris
```

```
- 確認發送時間（建議：週日晚上或週一早上）
- 最後拼字檢查
- 發送！
```

## ⚠️ 常見問題

### Q1: 我找不到足夠的資料怎麼辦？
```
A: 
1. 確認這個案例夠大嗎？太小的交易可能資料不足
2. 試著搜尋：
   - 英文 + 中文關鍵字
   - 公司併購後的年報（常有整合細節）
   - 學術論文（以案例研究形式）
3. 實在找不到 → 換案例
   （資料可獲得性是選題評分標準之一）
```

### Q2: 我寫得太學術/正式怎麼辦？
```
A: 
用「朋友測試」：
想像你在和朋友解釋這個案例（朋友懂基本商業概念）
你會怎麼說？

技巧：
- 用「你」而非「讀者」
- 用問句引導「為什麼？」
- 用比喻「這就像...」
- 用短句製造節奏
```

### Q3: 我寫超過5000字了怎麼辦？
```
A: 
砍掉這些：
- 重複的例子（留最好的那個）
- 過於細節的財務數據（留關鍵數字即可）
- 過長的背景鋪陳（聚焦在「為什麼重要」）

保留這些：
- 獨特的洞察
- 戲劇性場景
- 金句
- 讓人印象深刻的數據
```

### Q4: 我不確定某個資訊是否準確怎麼辦？
```
A: 
1. 找第二個來源驗證
2. 如果找不到第二個來源：
   - 如果是核心事實（價格、日期）→ 不要用
   - 如果是次要細節 → 加上「據報導」「傳聞」等字眼
3. 當有疑慮時，寧可刪除也不要冒險
```

### Q5: 近期鉤子找不到怎麼辦？
```
A: 
替代方案：
1. 用產業趨勢當鉤子
   「最近很多科技公司都在談AI併購，但25年前...」
   
2. 用週年紀念
   「今天是X案例的20週年...」
   
3. 用對比
   「當我們慶祝Y成功，很少人記得X的失敗...」

實在找不到 → 直接從歷史案例開篇也OK
（偶爾為之，不要每次都這樣）
```

## 🎯 成功標準

### 一篇好文章應該：

**內容層面：**
- [ ] 讀者學到了官方說法背後的真相
- [ ] 有至少1個「原來如此」的時刻
- [ ] 連接了過去與現在

**寫作層面：**
- [ ] 讀起來輕鬆但不膚淺
- [ ] 有故事性（不是教科書）
- [ ] 有明確觀點（不是中立報導）

**技術層面：**
- [ ] 所有事實可驗證
- [ ] 字數3000-5000
- [ ] 引用來源完整
