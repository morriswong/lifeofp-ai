# 雪花法：長篇文章設計模板

改編自 Randy Ingermanson 的 Snowflake Method，專為「以書／人物為切入點，探討更大命題」的長篇文章而設。

---

## 原理

雪花法的核心：從最小的結構開始，逐層擴展，直到你擁有一篇完整的文章。

小說的雪花法是「三場災難加結局」。文章的雪花法是「一個張力，三個轉折，一個落點」。每一步都建立在上一步之上，但你隨時可以回頭修改前面的步驟。

---

## 第一步：一句話（30 分鐘）

用一句話寫出你篇文章的核心張力。

不是主題，不是論點，是張力——一個讓讀者覺得「點解會咁？」的矛盾或反差。

**規則：**
- 少於 25 個字
- 必須包含一個矛盾、意外、或反直覺的元素
- 不要用人名，用身份描述（「史上最成功的投資者」而非「Warren Buffett」）

**範例：**
> 全球最大的健身連鎖店，靠的是會員不來健身。

> 教全世界寫作的人，自己從來不列大綱。

> 發明了效率管理的人，每天只工作四小時。

**自問：** 如果有人在電梯裡問你「你篇文寫乜？」，你會點答？

---

## 第二步：五句話（1 小時）

將核心張力擴展為五句話，對應文章的五個結構單元：

| 句子 | 功能 | 說明 |
|------|------|------|
| 第一句 | **開場設定** | 建立背景，製造與核心張力的反差 |
| 第二句 | **轉折一** | 引入人物或故事，提供意料之外的脈絡 |
| 第三句 | **轉折二** | 人物的核心觀點或發現，挑戰讀者的既有認知 |
| 第四句 | **轉折三** | 將觀點推到極致，或揭示更深層的問題 |
| 第五句 | **落點** | 拉回現實，讓讀者帶著一個問題離開 |

**範例 A（商業類）：**
1. 全球健身產業年收入超過 960 億美元，但商業模式的核心假設是：大部分會員不會來。
2. Planet Fitness 的創辦人不是健身狂，而是一個觀察到健身房令普通人恐懼的局外人。
3. 他的策略——讓健身房盡量不像健身房——違反了行業的每一條常識。
4. 但這個模式的真正風險不是競爭對手，而是如果會員真的開始來，一切就會崩塌。
5. 今天每當你看到一張從未用過的健身卡，你就見證了一個價值千億的心理學實驗。

**範例 B（科技／文化類）：**
1. 維基百科是全球訪問量第五高的網站，但沒有一個員工負責寫內容。
2. 創辦人原本在做一個由專家撰寫的線上百科全書，失敗後才意外轉向眾包模式。
3. 所有人都說匿名編輯會導致混亂，但維基百科的準確率在研究中反覆被證明接近專業百科。
4. 真正的威脅不是錯誤資訊，而是編輯社群的萎縮——越來越少人願意無償貢獻。
5. 你每天都在用一個理論上不應該運作的東西，而它正在慢慢失去讓它運作的人。

---

## 第三步：概念線索表（1-2 小時）

文章不是靠「角色」推動，而是靠「概念線索」。每條線索是一個貫穿全文的想法，在不同段落出現、發展、變化。

為每條主要線索填寫：

### 線索名稱：___________

- **一句話定義：** 這條線索是什麼？
- **在文章中的角色：** 它支撐核心張力的哪一面？
- **出場點：** 第一次出現在哪裡？用什麼方式引入？
- **發展：** 它在文章中段如何深化或轉變？
- **落點：** 它最後怎樣收束？讀者記住的是什麼？

**範例——「隱形成本」線索：**
- 一句話定義：每個看似免費的服務，都有一個用戶看不見的代價
- 角色：支撐核心張力中「免費的東西最貴」那一面
- 出場點：在講述商業模式起源的段落首次出現，用一個具體的用戶故事引入
- 發展：從個人案例擴展到行業數據，再引入專家的反對觀點
- 落點：結尾回到開頭的用戶，讓讀者意識到自己也是同樣處境

**常見線索類型：**
- 人物生平線（傳記式文章的骨架）
- 核心論點線（你要讀者記住的一個觀點）
- 反面案例線（用來反襯核心論點）
- 時代背景線（將個人故事連接到更大的趨勢）
- 回扣線（開頭拋出、結尾收回的元素）

---

## 第四步：段落擴展（2-3 小時）

將第二步的五句話，每句擴展為一段（3-5 句）。

**每段必須回答：**
1. 這一段要讀者知道什麼？（資訊）
2. 這一段要讀者感受什麼？（情緒）
3. 讀完這一段，讀者為什麼要繼續讀下一段？（動力）

**轉折段的特別要求：**
每個轉折段應該改變讀者的理解。讀完之前和讀完之後，讀者對主題的看法應該不同。如果沒有改變，這不是轉折，只是資訊堆疊。

---

## 第五步：證據與故事配對（2-3 小時）

為每個段落找到至少一個：

| 類型 | 說明 | 效果 |
|------|------|------|
| **具體事例** | 真實發生的事件，有人名、時間、地點 | 建立可信度 |
| **數字** | 統計、金額、百分比 | 製造衝擊 |
| **引用** | 當事人的原話 | 提供聲音和個性 |
| **類比** | 用讀者熟悉的事物解釋陌生概念 | 幫助理解 |

**鐵則：** 不要同時用兩個抽象論點。每個論點之後必須跟一個具體的東西——數字、故事、引用、畫面。抽象 → 具體 → 抽象 → 具體，交替進行。

**範例：**
- 論點：用戶其實不想要更多選擇（抽象）
- 跟隨：超市果醬實驗——24 種口味的攤位試吃人多，但 6 種口味的攤位購買率高十倍（具體）
- 延伸：選擇過多導致決策癱瘓的心理機制（抽象）
- 跟隨：Netflix 的首頁永遠只推薦一部片，不是因為技術限制，是刻意設計（具體畫面）

---

## 第六步：開頭與結尾設計（1-2 小時）

### 開頭的工作

開頭不是「介紹主題」。開頭是製造一個缺口——讓讀者腦中產生一個問題，然後他必須讀下去才能填補。

**三種有效的開場：**

1. **數字反差：** 用一個巨大的數字，然後立刻質疑它背後的前提
   > 全球每年丟掉 13 億噸食物。問題不是生產不夠，是分配從一開始就壞了。

2. **身份矛盾：** 一個人的身份和他的行為互相矛盾
   > 全世界最著名的極簡主義倡導者，家裡有三個儲物倉庫。

3. **時間錯位：** 一個很久以前的觀察，在今天依然成立（甚至更準確）
   > 一個經濟學家在 1930 年預測人類會因為太閒而痛苦。九十年後，我們比任何時代都忙，但他說對了一半。

### 結尾的工作

結尾不是「總結全文」。結尾是將文章的能量轉移到讀者的日常生活中。

**有效結尾的公式：** 具體場景 + 一個讀者可以自己回答的問題

> 今晚打開雪櫃的時候，數一下有幾樣東西會在你吃之前過期。那個數字，就是這篇文章在講的所有問題的縮影。

> 下次開會的時候，留意一下誰在說話、誰在沉默。你會發現，決定結果的往往不是內容，是座位。

---

## 第七步：段落順序與過渡（1-2 小時）

用試算表或列表排出所有段落，每段一行：

| # | 段落主題 | 線索 | 開頭句（草稿） | 結尾如何帶到下一段 |
|---|---------|------|---------------|------------------|
| 1 | | | | |
| 2 | | | | |
| ... | | | | |

**過渡的原則：**
- 最好的過渡是隱形的——上一段的結尾自然引出下一段的問題
- 如果你需要用「但是」、「然而」、「另一方面」來連接兩段，代表它們之間缺少邏輯橋樑
- 每個段落的第一句，應該讓人想起上一段最後留下的問題

---

## 第八步：標題（最後做）

鐵則：至少寫 16 個版本。

**標題的功能不是描述內容，是製造好奇。**

**自我測試：**
- 如果刪掉標題，只看正文第一段，讀者會不會失去什麼？如果不會，標題沒有在工作。
- 標題有沒有包含一個矛盾、意外、或具體的畫面？
- 讀者看完標題之後，腦中是否會產生一個問題？

---

## 檢查清單（完稿前）

- [ ] 核心張力是否在前三段內建立？
- [ ] 每個轉折是否真的改變了讀者的理解？
- [ ] 抽象與具體是否交替出現？（連續兩段抽象 = 危險信號）
- [ ] 每個段落是否都有往下讀的動力？
- [ ] 開頭是否製造了一個缺口？
- [ ] 結尾是否把能量交還給讀者？
- [ ] 是否至少寫了 16 個標題版本？
- [ ] 大聲讀一遍——節奏是否自然？

---

## 時間預算

| 步驟 | 預計時間 |
|------|---------|
| 第一步：一句話 | 30 分鐘 |
| 第二步：五句話 | 1 小時 |
| 第三步：概念線索表 | 1-2 小時 |
| 第四步：段落擴展 | 2-3 小時 |
| 第五步：證據與故事配對 | 2-3 小時 |
| 第六步：開頭與結尾設計 | 1-2 小時 |
| 第七步：段落順序與過渡 | 1-2 小時 |
| 第八步：標題 | 1 小時 |
| **設計總計** | **約 10-15 小時** |
| 正式撰寫 | 另計 |

設計完成後，你手上應該有：一個核心張力、一個五段結構、若干概念線索、每段的證據配對、一個經過設計的開頭和結尾、至少 16 個標題版本。

然後，寫。
